<task type="auto">
  <name>Task 1: Build seating configuration UI and Firestore persistence</name>
  <files>public/index.html</files>
  <action>
Populate the dashboard-seating section (created by Plan 01) with the seating configuration UI.

**Seating config data model** (stored in Firestore at teachers/{uid}/config/main, merged with existing kioskPin/avoidPairs/frontRow):

```javascript
seatingConfig: {
  enabled: true,              // false = no seat assignment at all
  numGroups: 6,               // number of groups (default 6)
  defaultSeatsPerGroup: 4,    // default seats per group (applied to all)
  perGroupOverrides: {        // optional per-group seat count overrides
    "3": 5,                   // group 3 gets 5 seats instead of 4
  },
  perPeriodOverrides: {       // optional per-period config
    "2": { enabled: false },  // period 2 has no seating
    "5": { numGroups: 8, defaultSeatsPerGroup: 3 }
  }
}
```

**UI layout:**

1. **Global seating toggle**: Switch or checkbox to enable/disable seating assignment entirely. When off, students see "Signed in!" without group/seat info. Clear visual state.

2. **Default configuration** (when enabled):
   - Number of groups: numeric input (1-50, default 6)
   - Default seats per group: numeric input (1-10, default 4)
   - Helper text: "For individual desk assignment, set groups to your number of desks and seats per group to 1"
   - Visual preview: simple grid showing groups and their seat counts

3. **Per-group adjustments**:
   - After setting defaults, show a list of groups with their seat count
   - Each group has an editable seat count (defaults to the default value)
   - Only show overrides that differ from default (clean UI)

4. **Per-period overrides** (expandable/collapsible section):
   - Header: "Period-specific settings (optional)"
   - List of periods from the teacher's roster (read from allRosters keys)
   - Each period can: use defaults, turn off seating, or set custom groups/seats
   - Only periods with overrides are saved

5. **Save button**: Saves seatingConfig to Firestore config/main (merge). Show success/error feedback.

6. **Load on init**: When dashboard loads, read seatingConfig from Firestore config/main and populate the UI. If no config exists, use reasonable defaults (enabled: true, 6 groups, 4 seats).

**Also move the existing "Seating Exceptions" section** (front row IDs, avoid pairs) from the Attendance tab into the Seating tab. It belongs here conceptually. Remove it from the attendance section and add it as a collapsible section in the Seating tab.
