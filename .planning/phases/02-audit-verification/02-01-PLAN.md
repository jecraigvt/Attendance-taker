Add two new functions for audit logging:

1. `log_sync_intent(student_id, period, intended_status, source_status, timestamp)` - Logs BEFORE attempting checkbox interaction
   - intended_status: What we plan to set (Absent, Tardy, Present)
   - source_status: Original status from Firebase/CSV (e.g., "Late", "On Time")
   - Writes to sync_audit_{YYYY-MM-DD}.log (daily file, not monthly like errors)

2. `log_sync_action(student_id, period, action_taken, checkbox_state, success, timestamp)` - Logs AFTER checkbox interaction
   - action_taken: What we did (checked_absent, unchecked_absent, checked_tardy, unchecked_tardy, no_change, skipped_locked)
   - checkbox_state: Final state {absent_checked: bool, tardy_checked: bool}
   - success: Whether the action completed without error

3. Add constant `AUDIT_LOG_FILE_TEMPLATE = "sync_audit_{date}.log"` for the daily audit file

Log format: JSON lines (consistent with existing error log format)
Example entry:
```json
{"type": "intent", "timestamp": "2026-02-05T10:30:45.123456", "student_id": "123456", "period": "3", "intended_status": "Tardy", "source_status": "Late"}
{"type": "action", "timestamp": "2026-02-05T10:30:45.456789", "student_id": "123456", "period": "3", "action_taken": "checked_tardy", "checkbox_state": {"absent": false, "tardy": true}, "success": true}
```

  <files>attendance-sync/upload_to_aeries.py</files>

Modify the student processing loop in upload_to_aeries.py to call audit logging functions:

1. Add imports at top:
```python
from sync_utils import log_sync_intent, log_sync_action
```

2. BEFORE checkbox interaction (after status normalization, before any checkbox logic):
```python
log_sync_intent(
    student_id=student_id,
    period=period,
    intended_status=status,  # Normalized status (Absent, Tardy, Present)
    source_status=raw_status,  # Original from CSV
    timestamp=datetime.now()
)
```

3. CAPTURE PRE-CHANGE CHECKBOX STATE (immediately before any checkbox clicks):
```python
# Read current checkbox state BEFORE making changes
was_already_absent = absent_box.is_checked() if absent_box else False
was_already_tardy = tardy_box.is_checked() if tardy_box else False
```
This is CRITICAL - we need to distinguish "checked_absent" (we made a change) from "no_change" (already in desired state).

4. AFTER checkbox interaction (in each checkbox logic branch):

For Absent branch (uses was_already_absent captured in step 3):
```python
log_sync_action(
    student_id=student_id,
    period=period,
    action_taken='checked_absent' if not was_already_absent else 'no_change',
    checkbox_state={'absent': True, 'tardy': False},
    success=True,
    timestamp=datetime.now()
)
```

For Tardy branch (uses was_already_tardy captured in step 3):
```python
log_sync_action(
    student_id=student_id,
    period=period,
    action_taken='checked_tardy' if not was_already_tardy else 'no_change',
    checkbox_state={'absent': False, 'tardy': True},
    success=True,
    timestamp=datetime.now()
)
```

For Present branch (correction logic):
```python
log_sync_action(
    student_id=student_id,
    period=period,
    action_taken='corrected_to_present',
    checkbox_state={'absent': False, 'tardy': False},
    success=True,
    timestamp=datetime.now()
)
```

5. For locked students (where we skip):
```python
log_sync_action(
    student_id=student_id,
    period=period,
    action_taken='skipped_locked',
    checkbox_state={'absent': None, 'tardy': None},
    success=True,
    timestamp=datetime.now()
)
```

6. In the exception handler for student processing failures:
```python
log_sync_action(
    student_id=student_id,
    period=period,
    action_taken='failed',
    checkbox_state={'absent': None, 'tardy': None},
    success=False,
    timestamp=datetime.now()
)
```
