## Edge Cases from Original Analysis

| Edge Case | Old Behavior | New Behavior | Improvement |
|-----------|--------------|--------------|-------------|
| <5 students in period | All "On Time" | Uses bell time | Small classes now have tardies |
| First 5 students arrive late | All "On Time", wrong threshold | Uses bell time | Actually late students marked late |
| 5th student very early | Wrong threshold, on-time marked late | Uses bell time | On-time students stay on-time |
| No bell schedule reference | Ignored actual start time | Uses actual start time | Matches school policy |

## Projected Dispute Impact

Based on PROJECT.md data:
- 15 of 29 corrections (52%) were tardy disputes
- Students disputed tardies signed in 8:27-8:34 AM for 8:30 class

With new logic (5-minute grace = 08:35 threshold):
- 8:27 sign-in = On Time (before bell)
- 8:30 sign-in = On Time (at bell)
- 8:34 sign-in = On Time (within grace)
- 8:35 sign-in = On Time (at boundary)
- 8:36+ sign-in = Late

**Projected result:** Most disputed tardies (8:27-8:34) would now be "On Time"
**Estimated dispute reduction:** 10-12 of 15 tardy disputes (67-80% reduction)
**Overall correction reduction:** From 29 to ~17-19 (35-40% reduction)

## Recommendations

1. **Monitor first week:** Track tardy counts vs historical data
2. **Adjust grace period if needed:** TARDY_GRACE_MINUTES can be changed
3. **Consider per-period grace:** Some periods might need different thresholds
4. **Document policy:** Make grace period visible to students/parents
