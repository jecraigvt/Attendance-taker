Add new functions to the existing sync_utils.py module:

1. **SELECTOR_STRATEGIES dict** - Define fallback selectors for critical elements:
   ```python
   SELECTOR_STRATEGIES = {
       'student_cell': [
           "td[data-studentid='{student_id}']",           # Primary: data attribute
           "td:has-text('{student_id}')",                 # Fallback 1: text content
           "//td[contains(@id, '{student_id}')]",         # Fallback 2: XPath id contains
       ],
       'absent_checkbox': [
           "span[data-cd='A'] input",                     # Primary: data-cd attribute
           "input[type='checkbox'][name*='Absent']",      # Fallback 1: name contains
           "span:has-text('A') input[type='checkbox']",   # Fallback 2: text label
       ],
       'tardy_checkbox': [
           "span[data-cd='T'] input",                     # Primary: data-cd attribute
           "input[type='checkbox'][name*='Tardy']",       # Fallback 1: name contains
           "span:has-text('T') input[type='checkbox']",   # Fallback 2: text label
       ],
       'period_dropdown': [
           "select",                                      # Primary: any select
           "select[id*='Period']",                        # Fallback 1: id contains Period
           "//select[contains(@name, 'Period')]",         # Fallback 2: XPath name contains
       ],
   }
   ```

2. **find_element_with_fallback function**:
   - Parameters: page, element_type (key from SELECTOR_STRATEGIES), format_args (dict for string formatting, e.g., student_id)
   - Try each selector strategy in order
   - On success: return (element, strategy_index)
   - If strategy_index > 0: log WARNING "Using fallback selector {index} for {element_type} - Aeries UI may have changed"
   - If all fail: raise SyncError with error_type='selector_failed'

3. **Alert logging**: When fallback is used (strategy_index > 0), also write to a separate alerts file:
   - File: selector_alerts_{YYYY-MM}.log
   - Format: timestamp, element_type, fallback_index, selector_used
   - This lets admin know the UI changed and primary selectors need updating
