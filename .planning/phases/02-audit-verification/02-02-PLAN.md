<task type="auto">
  <name>Task 1: Add verification report generator to sync_utils.py</name>
  <files>attendance-sync/sync_utils.py</files>
  <action>
Add function to generate verification report by comparing CSV source data with audit log.

**Architecture Note:** The CSV file IS the authoritative Firebase export - it is generated fresh each sync by attendance_to_aeries.py directly from Firebase data. Therefore, comparing CSV to audit log IS comparing Firebase source data to sync actions. No separate Firebase query is needed.

`generate_verification_report(csv_filepath, run_start_timestamp, output_dir=".")` -> dict

The function should:

1. Read the source CSV file to get expected actions (what we should have synced)
2. Read audit log entries from run_start_timestamp using get_sync_run_entries()
3. For each student in CSV, compare:
   - Expected: student_id, period, status (from CSV)
   - Actual: Look up matching audit entries (intent + action)

4. Identify discrepancies:
   - **missing_intent**: Student in CSV but no intent logged (processing was skipped entirely)
   - **missing_action**: Intent logged but no action logged (processing started but didn't complete)
   - **status_mismatch**: Intent logged with different status than CSV (normalization bug)
   - **action_failed**: Action logged with success=False

5. Generate report dict:
```python
{
    "timestamp": "2026-02-05T15:30:45",
    "csv_file": "attendance_2026-02-05.csv",
    "summary": {
        "total_students": 150,
        "total_synced": 148,
        "total_failed": 2,
        "total_skipped_locked": 5,
        "total_discrepancies": 2
    },
    "by_period": {
        "1": {"synced": 25, "failed": 0, "locked": 1},
        "2": {"synced": 28, "failed": 1, "locked": 0},
        ...
    },
    "discrepancies": [
        {
            "type": "missing_action",
            "student_id": "123456",
            "period": "3",
            "expected_status": "Tardy",
            "actual": "No action logged"
        },
        ...
    ]
}
```

6. Write report to file:
   - Filename: sync_verification_{YYYY-MM-DD}_{HHMMSS}.txt
   - Format: Human-readable text with sections for Summary, By Period, and Discrepancies
   - Also write JSON version: sync_verification_{YYYY-MM-DD}_{HHMMSS}.json

7. Return the report dict for programmatic use

```python
logger.info("")

    report = generate_verification_report(
        csv_filepath=csv_file,
        run_start_timestamp=sync_start_time,
        output_dir="."
    )

    # Log summary
    summary = report['summary']
    logger.info(f"   Total students: {summary['total_students']}")
    logger.info(f"   Synced: {summary['total_synced']}")
    logger.info(f"   Failed: {summary['total_failed']}")
    logger.info(f"   Skipped (locked): {summary['total_skipped_locked']}")

    if summary['total_discrepancies'] > 0:
        logger.warning(f"   !! DISCREPANCIES FOUND: {summary['total_discrepancies']}")
        for disc in report['discrepancies'][:5]:  # Show first 5
            logger.warning(f"      - {disc['type']}: Student {disc['student_id']} Period {disc['period']}")
        if len(report['discrepancies']) > 5:
            logger.warning(f"      ... and {len(report['discrepancies']) - 5} more (see report file)")
    else:
        logger.info("   No discrepancies found")

except Exception as e:
    logger.error(f"   Failed to generate verification report: {e}")
```
