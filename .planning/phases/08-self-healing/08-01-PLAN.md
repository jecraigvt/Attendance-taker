**Create `railway-worker/healer.py`** with an `attempt_heal()` function. This is the core self-healing module.

The function signature:
```python
def attempt_heal(
    page,              # Playwright page (already on the Aeries attendance page)
    element_type: str, # e.g. "absent_checkbox", "student_cell"
    format_args: dict, # e.g. {"student_id": "12345"} — same args passed to find_element_with_fallback
    original_selectors: list,  # The static selectors that already failed
    teacher_uid: Optional[str] = None,  # For healing event logging context
) -> Optional[str]:
    """
    Attempt to heal a broken selector using Gemini LLM.
    Returns a working CSS/XPath selector string, or None if healing failed.
    """
```

**Implementation details:**

1. **Daily cap check** — Call `get_healing_call_count_today()` from firestore_client. If >= 25, log "Daily healing cap reached" and return None immediately. Do NOT count cap-blocked attempts as Gemini calls.

2. **DOM context preparation** — Extract relevant DOM context from the Playwright page for Gemini:
   - For `student_cell`: Get the attendance table HTML. Use `page.locator("table").first.inner_html()` (or a similar broad locator). If the table is huge (>50KB), truncate to the first 30KB to stay within Gemini's input limits.
   - For `absent_checkbox` / `tardy_checkbox`: Get the parent row's HTML (the `format_args` won't have `student_id` at this point since these are called within a row context — pass the row HTML as an optional parameter, or get `page.content()` truncated).
   - For `period_dropdown`: Get the page's `<select>` elements HTML.
   - General approach: Use `page.content()` truncated to ~30KB as the DOM context. Simplify by stripping `<script>` and `<style>` tags to reduce noise.

3. **Gemini prompt** — Construct a prompt that:
   - Describes what element type we need (e.g., "the checkbox that marks a student as Absent")
   - Provides the original selectors that used to work as examples of what we're looking for
   - Includes the extracted DOM context
   - Asks Gemini to return ONLY a single CSS selector (or XPath prefixed with `xpath=`) that would match the target element
   - Instructs Gemini to return "NONE" if it cannot identify the element

   Example prompt structure:
   ```
   You are helping repair a broken CSS selector for a school attendance web application (Aeries SIS).

   I need to find: {description for element_type}

   These selectors USED to work but no longer match any elements:
   {original_selectors as bulleted list}

   Here is the current page HTML:
   ```html
   {dom_context}
   ```

   Return ONLY a single CSS selector (or XPath prefixed with "xpath=") that matches the target element. If you cannot identify it, return exactly "NONE".
   ```

   Element type descriptions:
   - `student_cell`: "A table cell (td) containing or associated with student ID {student_id}"
   - `absent_checkbox`: "The checkbox input for marking a student Absent (typically labeled 'A')"
   - `tardy_checkbox`: "The checkbox input for marking a student Tardy (typically labeled 'T')"
   - `period_dropdown`: "A dropdown (select element) for choosing which class period to view"

4. **Gemini Flash call** — Use `google.generativeai` library:
   ```python
   import google.generativeai as genai
   genai.configure(api_key=os.environ.get("GEMINI_API_KEY"))
   model = genai.GenerativeModel("gemini-2.0-flash")
   response = model.generate_content(prompt)
   candidate_selector = response.text.strip()
   ```
   Wrap in try/except. On any exception (API error, timeout, quota), log the error and fall through to Pro.

5. **Selector validation** — After getting a candidate selector from Gemini:
   - If candidate is "NONE" or empty, skip to Pro (or return None if already on Pro)
   - Format the candidate with `format_args` (same `.format(**format_args)` as the static selectors)
   - Try `page.locator(formatted_candidate).count()` — if > 0, the selector is valid
   - If count is 0, the selector doesn't match — skip to Pro (or return None if on Pro)

6. **Gemini Pro escalation** — If Flash returns "NONE", returns an invalid selector, or throws an error:
   - Repeat the same prompt with `gemini-2.0-pro` model (not `gemini-1.5-pro` — use the latest)
   - Same validation logic
   - If Pro also fails, return None

7. **Firestore logging** — After each healing attempt (success or failure), call `write_healing_event()`:
   ```python
   write_healing_event(
       element_type=element_type,
       original_selectors=original_selectors,
       healed_selector=candidate_selector if success else None,
       model_used="flash" or "pro",
       success=True/False,
       teacher_uid=None,  # optional, for context
   )
   ```

8. **Return value** — Return the validated selector string on success, or None on failure.

**Add to `railway-worker/firestore_client.py`** two new functions:

```python
def write_healing_event(
    element_type: str,
    original_selectors: list,
    healed_selector: Optional[str],
    model_used: str,
    success: bool,
    teacher_uid: Optional[str] = None,
) -> None:
    """
    Write a healing event to the global 'healing_events' collection.

    Path: healing_events/{auto-id}
    Fields: element_type, original_selectors, healed_selector, model_used,
            success, teacher_uid, timestamp (SERVER_TIMESTAMP)
    """
```

```python
def get_healing_call_count_today() -> int:
    """
    Count healing events written today (UTC) in the global 'healing_events' collection.

    Query: healing_events where timestamp >= start_of_today_utc
    Returns the count. On error, returns 0 (fail-open: allow healing on read errors).
    """
```

The daily cap query uses a Firestore compound query filtering by timestamp. Keep the query efficient — the healing_events collection will be small (max 25/day).

**Important design decisions:**
- Healed selectors are used for the CURRENT sync only (not persisted to selectors.json). If the same selector breaks again next cycle, Gemini re-heals with fresh page context. This avoids stale healed selectors and keeps selectors.json as the developer-maintained baseline.
- The `attempt_heal` function is called with the Playwright page already on the correct Aeries attendance page — no navigation needed.
- GEMINI_API_KEY is read from environment variable (consistent with FERNET_KEY pattern). It is NOT required — if missing, healing is silently disabled (log a warning on first attempt, then skip).
