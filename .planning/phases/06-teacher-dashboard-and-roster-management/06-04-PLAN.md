<task type="auto">
  <name>Task 1: Build onboarding wizard</name>
  <files>public/index.html</files>
  <action>
Create a step-by-step onboarding wizard for first-time teachers. The wizard appears after login for teachers who haven't completed setup.

**Wizard trigger logic** — After successful login (in handleLogin):
1. Check if teacher needs onboarding: read `teachers/{uid}/config/main` from Firestore
2. If `onboardingComplete: true` exists, skip wizard — go to PIN setup (if no PIN) or kiosk mode
3. If no config or `onboardingComplete` is missing/false, show onboarding wizard
4. Existing users (Jeremy) should NOT be re-onboarded — check if they have roster data or config

**Wizard structure** — Full-screen overlay similar to login screen:

**Step 1: Credentials Verified (confirmation)**
- "Your Aeries credentials have been verified!" (they just logged in)
- Show username (from login)
- "Next" button to proceed
- Progress indicator: "Step 1 of 3"

**Step 2: Roster Setup**
- "Let's fetch your class roster from Aeries"
- "Fetch Roster" button — calls fetchRoster Cloud Function
- Loading state with spinner
- On success: show period/student count summary, "Next" button
- On failure (roster_requires_browser or error): show "We couldn't automatically fetch your roster. You can upload a CSV file instead." with CSV upload control (reuse existing upload logic). After upload, "Next" button
- Option to "Skip for now" (can upload roster later from dashboard)
- After roster fetch/upload: show preferred names review step
  - List students by period with editable preferred name fields
  - "These names will be shown to students on the sign-in screen"
  - "Save & Continue" button
- Progress indicator: "Step 2 of 3"

**Step 3: Seating Configuration**
- "How do you want to assign seats?"
- Three clear options presented as cards:
  A. **Group seating** — "Students are assigned to groups with multiple seats" (default: 6 groups, 4 seats each)
  B. **Individual desks** — "Each student gets a numbered desk" (= groups of size 1)
  C. **No seating** — "Students just sign in without seat assignment"
- If A or B selected: show quick config controls (number of groups, seats per group)
- Helper text for B: "Tip: Set seats per group to 1 for individual desks"
- "Finish Setup" button — saves seatingConfig to Firestore, sets `onboardingComplete: true`
- Progress indicator: "Step 3 of 3"

**After wizard completes:**
- Save `onboardingComplete: true` to config/main
- If no PIN set yet: show PIN setup screen
- If PIN exists: transition to dashboard (NOT kiosk — CONTEXT decision: "After onboarding, teacher lands on their dashboard")

**Visual design:**
- Full-screen overlay on dark background (consistent with login screen aesthetic)
- Clean white card centered on screen
- Progress bar or step dots at top
- Tailwind styling consistent with existing app (red-700 accent, gray-50 backgrounds)
  </action>

3. Wizard appears with Step 1 (credentials confirmed)

6. Step 3 (seating) appears with three options
7. Select group seating, configure, click Finish
8. Wizard closes, PIN setup appears (first time) or dashboard appears
9. Sign out and sign back in — wizard does NOT appear (onboardingComplete: true)
  </verify>
  <done>Onboarding wizard with 3 steps (credentials, roster, seating). Progress indicator. First-time teachers guided through setup. Returning teachers skip wizard.</done>
