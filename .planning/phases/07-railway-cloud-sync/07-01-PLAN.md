  <action>
Create a `railway-worker/` directory at the project root.

**railway-worker/Dockerfile:**
Use `mcr.microsoft.com/playwright/python:v1.49.1-noble` as the base image (Microsoft's official Playwright Docker image with browsers pre-installed). This avoids the most common Playwright-in-Docker failure: missing system libraries for Chromium.

Steps in Dockerfile:
1. `FROM mcr.microsoft.com/playwright/python:v1.49.1-noble`
2. Set `WORKDIR /app`
3. Copy `requirements.txt` and run `pip install --no-cache-dir -r requirements.txt`
4. Run `playwright install chromium` (ensures correct Chromium version even if base image drifts)
5. Copy all Python source files
6. Set `ENV PYTHONUNBUFFERED=1` for clean logging
7. `CMD ["python", "worker.py"]` (worker.py will be created in Plan 07-02; for now the smoke test validates the image)

**railway-worker/requirements.txt:**
Pin these dependencies:
```
firebase-admin>=6.0.0,<7.0.0
playwright>=1.49.0,<2.0.0
cryptography>=41.0.0,<44.0.0
apscheduler>=3.10.0,<4.0.0
pytz>=2024.1
```

Notes:
- `cryptography` provides `cryptography.fernet.Fernet` which is cross-compatible with the Node `fernet` npm package (same Fernet spec).
- `apscheduler` for scheduled job execution (will be used in Plan 07-02).
- `pytz` for Pacific Time timezone handling.

**railway-worker/.dockerignore:**
```
__pycache__
*.pyc
.env
*.png
*.csv
*.log
*.txt
!requirements.txt
```

**railway.toml** (at project root):
```toml
[build]
dockerfilePath = "railway-worker/Dockerfile"

[deploy]
healthcheckPath = "/"
restartPolicyType = "ON_FAILURE"
restartPolicyMaxRetries = 3
```

Note: Railway auto-detects the Dockerfile. The `railway.toml` tells Railway to look in the `railway-worker/` subdirectory for the build context. Set `dockerfilePath` accordingly. Since this is a worker (not a web server), we may need to remove `healthcheckPath` or set `numReplicas` — Railway workers don't need health checks. Configure:
```toml
[build]
dockerfilePath = "railway-worker/Dockerfile"
builder = "DOCKERFILE"

[deploy]
numReplicas = 1
restartPolicyType = "ON_FAILURE"
restartPolicyMaxRetries = 5
```

  <name>Task 2: Create smoke test script that validates Playwright and Firebase in Docker</name>
  <files>

  <action>
Create `railway-worker/smoke_test.py` — a self-contained script that validates the Docker environment works correctly. This script will be used to verify the container on Railway before deploying the full sync worker.

The smoke test should:

1. **Test Playwright browser launch:**
   - Import `from playwright.sync_api import sync_playwright`
   - Launch headless Chromium
   - Navigate to `https://example.com` (lightweight, always-available page)
   - Assert the page title contains "Example"
   - Close browser
   - Print `[PASS] Playwright: Headless Chromium launched and navigated successfully`

2. **Test Firebase Admin SDK initialization:**
   - Read `FIREBASE_SERVICE_ACCOUNT` env var (JSON string, not file path)
   - Parse it with `json.loads()`
   - Initialize Firebase Admin with `credentials.Certificate(json.loads(env_var))`
   - Verify `firebase_admin.get_app()` succeeds
   - Print `[PASS] Firebase: Admin SDK initialized from environment variable`
   - Clean up the app with `firebase_admin.delete_app(app)`

3. **Test Fernet decryption cross-compatibility:**
   - Read `FERNET_KEY` env var
   - Create a `cryptography.fernet.Fernet(key)` instance
   - Encrypt a test string, then decrypt it back
   - Assert roundtrip works
   - Print `[PASS] Fernet: Encryption/decryption roundtrip successful`

4. **Summary:** Print pass/fail count and exit with code 0 if all pass, 1 if any fail.

Each test should be wrapped in try/except so one failure doesn't block the others. If an env var is missing, print `[SKIP] {test}: {env_var} not set` instead of failing.

The script should be runnable as: `python smoke_test.py`
  </action>

  </verify>
